# 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트 명**: ABC (Ad Banner Creator)
- **설명**: 심의 규정을 준수하면서 기존 광고 소재의 배리에이션(variation)을 자동으로 제작하고 관리하는 프로그램입니다.
- **핵심 가치**: 광고 배너 제작에 투입되는 리소스와 시간을 절감하고, 심의 준수 보장 및 에셋 히스토리 관리를 효율화합니다.

## 2. 타겟 사용자
- **주요 사용자**: 현대카드 임직원 (광고 담당 부서)

## 3. 문제 정의 및 목표
### 3.1 해결하려는 문제
- **A**: 디지털 웹/앱 광고 배너 제작 시 심의 규정을 준수하며 다양한 버전을 만드는 데 상당한 리소스가 투입되고 일정이 지연됨.
- **B**: 다양한 지면에서 사용된 콘텐츠들의 히스토리를 관리하고 추적하는 데 어려움이 있음.

### 3.2 목표
- 심의를 준수한 광고 배너 배리에이션 제작의 자동화.
- 과거 광고 에셋에 대한 체계적인 관리 및 조회 기능 제공.

## 4. 사용자 스토리
1. **효율성**: "배너에 활용할 수 있는 이미지를 빠르고 다양하게 만들고 싶다."
2. **심의 준수**: "선택된 템플릿을 바탕으로 심의 규정을 엄격히 준수한 완성본을 빠르게 제작하고 싶다."
3. **유연성**: "다양한 광고 배너를 운영하여 A/B 테스트 등을 유연하게 진행하고 싶다."
4. **히스토리 관리**: "이전에 사용된 광고 에셋들을 시간, 컬러, 지면 크기 등 다양한 기준으로 검색하고 싶다."

## 5. 화면 설계서

### 5.1 화면 1: 입력 (Banner Generation Setup)
**목적**: 심의 가이드라인, 문구, 템플릿 및 레퍼런스 이미지를 입력합니다.

- **레이아웃 컨셉**: 입력 작업에 집중할 수 있는 심플하고 여백이 충분한 구조.
- **UI 구성**:
    - **상단 (Header)**: 서비스 로고, 서비스 설명, 오늘 날짜.
    - **중간 (Body - 입력 필드)**:
        - **심의 가이드라인**: 심의 기준이 기재된 .txt 파일 업로드.
        - **활용 문구**: 이미 심의를 받은 문구들(~10개)이 포함된 .txt 파일 업로드.
        - **레퍼런스 배너** (선택 사항): 문구 및 템플릿을 바탕으로 작업된 레퍼런스 이미지 업로드.
    - **하단 (Footer)**: 서비스 문의처 정보.
- **기능**: 입력 받은 자료들을 조합하여 오픈ai API를 활용해 배너 이미지를 생성합니다.

### 5.2 화면 2: 출력 (Variation Review)
**목적**: 생성된 배너 결과물을 확인하고 속성별로 필터링합니다.

- **레이아웃 컨셉**: 정보 전달 위주의 단순한 인터페이스.
- **UI 구성**:
    - **상단 (Header)**: 입력 로그 (날짜 / 심의 정보 / 사용 문구 / 사용 템플릿).
    - **중간 (Body)**:
        - **생성된 배너**: 생성된 이미지들을 그리드 또는 리스트 형태로 표시.
        - **필터 기능**: 컬러 별로 드롭다운 필터링.
    - **하단 (Footer)**: "더보기" 버튼 또는 무한 스크롤.
- **기능**: 오픈ai API로 생성된 배너 결과물을 나열하고 컬러 별로 필터링합니다.


### 5.3 화면 3: 대시보드 (Asset Management)
**목적**: 심의 통과된 배너를 아카이빙하고 세부 정보를 관리합니다.

- **레이아웃 컨셉**: 가시성을 극대화한 데이터 그리드 스타일의 안정적인 읽기 전용 화면.
- **UI 구성**:
    - **입력 테이블**:
        - `입력 ID (Input ID)`
        - `심의 통과 배너 이미지`
        - `세부 요소` (문구, 분위기 태그)
        - `심사 일자`
    - **출력 테이블**:
        - `입력 ID (Input ID)` (외래 키)
        - `출력된 배너 이미지`
        - `세부 요소` (문구, 분위기 태그)
        - `채택 여부` (실제 활용 여부)
- **기능**: 지금까지 생성된 이미지를 모두 아카이빙하고, 세부 정보를 관리합니다.
